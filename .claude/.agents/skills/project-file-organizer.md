# 项目文件整理 Skill（完整版）

## 核心原则

1. **安全第一**：任何重命名、移动、删除操作前，必须列出清单并等待用户明确确认。
2. **默认归类**：除非用户明确反对，否则将同类型文件移动到对应的类型文件夹。
3. **笔记优先**：用户提到“笔记”、“帮我整理”等关键词时，生成的报告自动存入笔记文件夹。
4. **未知类型处理**：遇到不熟悉的文件类型，先询问用户意图；若用户拒绝，则移入自动生成的“垃圾桶文件夹”。
5. **最小权限原则**：文件权限只授予必要的最小范围。

## 触发条件与笔记文件夹

### 关键词触发
当用户请求包含“笔记”、“帮我整理”、“记录整理结果”、“保存整理记录”等词时，整理报告自动保存到笔记文件夹。

### 笔记文件夹确定顺序
1. 用户明确指定路径。
2. 环境变量 `NOTES_DIR`。
3. 常见默认路径（按序检测）：`./notes/` → `~/notes/` → `~/Documents/notes/` → `~/OneDrive/笔记/`。
4. 询问用户：“请问你的笔记文件夹路径是什么？”
5. 兜底：保存到当前目录下的 `file_organization_logs/`。

### 保存到笔记文件夹的内容
- 整理变更清单（执行前展示的列表）
- 执行后总结报告
- 目录结构快照（整理前后）
- 用户要求“记录下来”的任何内容

> 注意：被整理的项目文件本身保留在原项目目录，只有生成的辅助文件才存入笔记文件夹。

## 按类型自动归类规则（默认启用）

除非用户明确说“不要移动文件”或“保持原位置”，否则将以下文件移动到对应的文件夹：

| 文件类型（扩展名） | 目标文件夹 | 说明 |
|------------------|------------|------|
| `.js`, `.ts`, `.jsx`, `.tsx` | `src/` 或 `scripts/` | 已有 `src/` 则用 `src/`，否则创建 `scripts/` |
| `.py` | `src/` 或 `python/` | Python 源码 |
| `.java` | `src/main/java/` | Java 源码 |
| `.go` | `cmd/` 或 `pkg/` | 含 `main` 函数用 `cmd/`，否则 `pkg/` |
| `.rs` | `src/` | Rust 源码 |
| `.c`, `.cpp`, `.h` | `src/` 或 `c/` | C/C++ 源码 |
| `.sh`, `.bash`, `.zsh` | `scripts/` | Shell 脚本 |
| `.css`, `.scss`, `.less` | `styles/` 或 `src/styles/` | 样式文件 |
| `.html`, `.htm` | `public/` 或 `html/` | 静态页面 |
| `.json`, `.yaml`, `.yml`, `.toml` | `config/` | 配置文件（`package.json` 等根目录必需文件保留原位） |
| `.md`, `.txt`, `.rst` | `docs/` | 文档（`README.md` 保留根目录） |
| `.png`, `.jpg`, `.jpeg`, `.gif`, `.svg` | `assets/images/` 或 `images/` | 图片资源 |
| `.ttf`, `.woff`, `.woff2` | `assets/fonts/` | 字体文件 |
| `.pdf`, `.doc`, `.docx` | `docs/` 或 `assets/` | 文档附件 |
| `.log`, `.tmp`, `.cache` | **不移动，直接询问是否删除** | 临时文件 |
| `.pyc`, `.o`, `.class`, `.exe` | **不移动，直接询问是否删除** | 编译产物 |

### 例外情况
- 根目录必需的框架文件：`package.json`, `README.md`, `LICENSE`, `.gitignore`, `Dockerfile`, `Makefile`, `go.mod`, `Cargo.toml` 等。
- 用户明确要求保留原位置的文件。
- 已位于合理目录中的文件。

## 文件命名规范

| 语言/类型 | 命名风格 | 示例 |
|----------|---------|------|
| JavaScript/TS | kebab-case | `user-service.ts` |
| Python | snake_case | `user_service.py` |
| Java/Kotlin | PascalCase | `UserService.java` |
| C# | PascalCase | `UserService.cs` |
| Go | snake_case | `user_service.go` |
| Rust | snake_case | `user_service.rs` |
| CSS/SCSS | kebab-case | `main-layout.scss` |
| HTML | kebab-case | `about-page.html` |
| Shell | snake_case | `deploy_script.sh` |
| 目录 | kebab-case | `src/components/` |

## 文件权限设置

- **可执行权限（`chmod +x`）**：Shell 脚本、带 shebang 的入口脚本、二进制文件、Git 钩子。
- **不应可执行**：源代码、配置文件、文档、静态资源。
- **默认权限**：目录 `755`，普通文件 `644`，可执行文件 `755`，私密文件（如 `.env`）`600`。

## 不熟悉文件类型的处理流程

当遇到以下情况时，视为“不熟悉的文件类型”：
- 扩展名不在上述归类表中，且你（CC）不确定其用途。
- 用户要求保存一个生成的文件到本地，但你不知道应该存什么格式或位置。
- 文件内容看起来异常（如二进制但扩展名为 `.txt`）。

### 处理步骤

1. **向用户询问**：
   - “我遇到了不熟悉的文件类型：`xxx.abc`。请问这是什么类型的文件？你希望我如何处理它？（例如：移动到某个文件夹、重命名、删除、保留原位）”

2. **如果用户拒绝回答或说“随便/不管”**：
   - 将该文件**移动到“垃圾桶文件夹”**。
   - 垃圾桶文件夹路径：在当前工作目录下创建 `.file_trash_YYYYMMDD/`（例如 `.file_trash_20250422/`）。
   - 如果用户后续要求恢复，可以从那里找回。

3. **记录到整理报告**：
   - 在报告中注明哪些文件因类型未知且用户拒绝指定而被移入垃圾桶，以及垃圾桶的路径。

### 示例

**用户**：帮我整理当前目录。

**CC**（扫描后发现 `data.xyz` 文件）：
我发现了扩展名为 `.xyz` 的文件，不在我的已知类型列表中。请问这是什么类型的文件？你希望我：
- A) 移动到某个指定文件夹（请告知）
- B) 重命名为标准扩展名（请告知正确扩展名）
- C) 保留原位不动
- D) 删除/移入垃圾桶

**用户**：随便。

**CC**：
由于你未指定处理方式，我将把 `data.xyz` 移动到垃圾桶文件夹 `.file_trash_20250422/` 中。你可以随时从那里恢复。

已移动。是否继续其他整理操作？

## 执行流程（完整）

当用户要求整理项目文件时，按以下步骤执行：

1. **询问用户偏好**（首次或未定义时）：
   - 是否允许按类型自动移动文件？
   - 笔记文件夹路径（若触发关键词）。
   - 是否要处理不熟悉的文件类型（按上述流程）。

2. **分析现状**：
   - 运行 `ls -la` 或 `tree`。
   - 识别不符合规范的文件、目录、权限问题、未知类型。

3. **生成变更清单**（分类列出）：
   - 移动/归类
   - 重命名
   - 删除
   - 权限修改
   - 未知类型文件（单独列出，并标注“将询问”）

4. **请求确认**：
   - 展示清单，提示：“是否执行以上 X 项变更？请输入 `确认` 或 `y` 继续，或输入序号选择性执行。”

5. **执行操作**（用户确认后）：
   - 按清单逐项执行，每项输出状态。
   - 若遇到未知类型且尚未询问，暂停并执行“不熟悉文件类型的处理流程”。

6. **生成整理报告**：
   - 将变更清单、执行结果、最终目录树写入 Markdown 文件。
   - 文件名：`file_organization_YYYYMMDD_HHMMSS.md`。
   - 保存到笔记文件夹（按前述规则确定）。
   - 告知用户报告路径。

7. **询问是否打开报告**（可选）。

## 删除和移动的安全规则

- **禁止删除/移动**：`.git/` 目录、`node_modules/`（除非用户特别授权）、`.env` 等私密文件、用户最近 24 小时内修改过的文件。
- **删除前二次确认**：对于批量删除或删除目录，需再次询问：“即将删除 X，确认吗？”
- **移动前创建目标目录**：`mkdir -p`。
- **移动后检查空目录**：可询问用户是否删除原空目录。

## 用户覆盖优先级

- 用户明确说“不要移动 X” → 永远不移动该路径。
- 用户说“只整理命名” → 跳过所有移动操作。
- 用户可以在确认清单时取消勾选特定项。
